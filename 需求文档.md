- 目标：
  开发一个网页版黑匣子剧场（事故调查解谜游戏）Demo，玩家扮演航空事故调查者，通过分析线索找出事故原因，学习航空安全知识。

  技术栈：
  - 前端：React.js + Tailwind CSS 或 Material UI，使用 Framer Motion 实现动画与交互。
  - 后端：Python 3.12 + FastAPI。
  - Node 版本：24。
  - Python 虚拟环境：使用 uv 创建。
  - 数据库：SQLite。
  - 项目文件夹：
    - frontend/ （前端项目）
    - backend/ （后端项目）

  项目结构：
  frontend/
    src/
      components/        # UI组件，如线索面板、历史案例弹窗、评分面板
      pages/             # 页面组件，如游戏主页面、调查报告页面
      services/          # 调用后端 API 的逻辑
      assets/            # 图片、音频等媒体资源
    package.json
    tailwind.config.js 或 Material UI 配置
  backend/
    main.py             # 启动 FastAPI 应用
    models.py           # SQLite ORM模型：事故、线索、因果链、历史案例、玩家提交
    routes/             # API路由：获取事故数据、提交调查报告、评分
    schemas.py           # Pydantic模型
    database.py         # SQLite数据库初始化与连接
    requirements.txt
  根目录：
    README.md
    .gitignore

  功能与逻辑：
  1. 事故建模：
     - 每个事故包含：ID、标题、描述、因果链、线索集合、历史案例
     - 因果链格式：因素 → 影响 → 结果
     - 线索类型：文本、图片、音频
     - 历史案例：真实事故简述

  2. 线索交互：
     - 玩家点击线索 → 更新游戏状态 → 弹窗显示反馈
     - 正确发现因素 → 弹出历史案例
     - 错误发现因素 → 提示继续探索

  3. 评分系统：
     - 玩家提交调查报告
     - 对比玩家选择的因素与真实因果链
     - 输出分数和评价

  4. 前端交互：
     - 点击查看线索信息
     - 拖拽线索至分析板（可选）
     - 弹窗显示历史案例
     - 提交最终调查报告 → 调用后端评分
     - 响应式布局，兼容桌面和移动端浏览器

  5. 后端接口：
     - GET /accidents → 获取事故列表
     - GET /accidents/{id} → 获取单个事故详情，包括线索和因果链
     - POST /submit-report → 玩家提交调查报告，返回评分

  6. 数据库要求：
     - SQLite存储事故表、线索表、历史案例表、玩家提交表
     - 自动初始化数据库，如不存在则创建
     - 提供示例数据：至少3个事故案例，每个包含线索和历史案例（图片或音频可用占位符）

  7. 运行约束：
     - 前端通过 npm install 和 npm start 可直接运行
     - 后端通过虚拟环境启动 FastAPI：uv run backend/main.py 或 python main.py
     - 所有依赖在 requirements.txt 和 package.json 中定义
     - 项目模块化，方便添加新事故案例

  额外要求：
  - 每个模块代码可复用、可扩展
  - 前后端交互清晰，API调用逻辑完整
  - UI友好、美观，交互流畅
  - 提供完整示例数据和媒体占位符，便于演示
  - 项目可直接用于黑客松 Demo 演示
